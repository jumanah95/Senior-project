Problem Definition

Challenge 01
Rigid Assessment Formats
Traditional exams may not always capture the full extent of students’ understanding. Fixed question formats and limited coverage can make it challenging to evaluate all aspects of student knowledge.
Challenge 02
Manual Grading Burden
Manual grading requires significant time and effort, particularly in large classes, which can affect consistency and efficiency in evaluation.
02 / 14
02

Proposed Solution

Conversational Platform
Students interact through natural, dynamic dialogue rather than static question formats.
Rubric-Based Evaluation
System evaluates understanding using instructor-defined rubrics — not keyword matching.
Automated Feedback
Generates detailed, criterion-level feedback and reports automatically after each assessment.
Core Vision
Not just a chatbot — an interactive assessment engine that evaluates understanding through follow-up dialogue, producing clear reports for instructors.
03 / 14
03

Literature Review Gap

CBA Papers Dialogue assessment exists, but lacks a full scoring workflow for instructors.
Dialogue Tutoring Follow-up questions aid learning but do not provide complete assessment functionality.
Knowledge Tracing Shows learning levels effectively, but is not a full examination system.
Rubric Grading Scoring is clearer with rubrics, but usually implemented separately from dialogue assessment.
Gap identified — No single system combines interactive dialogue, rubric-based scoring, and instructor-ready reports. This project fills that gap.

04

Methodology

1
Instructor Setup — Defines assessment topics, rubric criteria, and scoring weights before the session begins.
2
Student Dialogue — Student answers questions through a natural chat interface at their own pace.
3
AI Interpretation — System interprets responses and generates targeted follow-up questions when answers are incomplete.
4
Understanding-Based Scoring — Scores reflect comprehension, not exact phrasing or keyword matching.
5
Final Report — After all topics, system generates a comprehensive report summarising the student's performance.

System Architecture

Frontend
User interface for both students and instructors — chat view and dashboard.
Backend
Handles all system logic, routing, and processing between components.
LLM API
Generates and analyses conversational responses for the assessment engine.
Database
Stores conversations, scores, feedback, and session transcripts.
Rubric Engine
Applies instructor-defined grading criteria to produce fair, consistent scores.
Instructor Dashboard
Displays results, reports, and analytics for each student session.

07

Limitations

LLM Interpretation Errors
Language models can misinterpret nuanced or ambiguous student responses, requiring oversight.
Rubric Quality Dependency
System performance depends heavily on how well the instructor defines the rubric criteria.
Limited Testing Scope
Current phase uses theoretical data only — real-world testing across subjects is not yet conducted.
Human Review Required
Edge cases and high-stakes assessments will still require instructor verification and review.
